How I Work

From messy operations to structured systems.

I don't start with code. I start with the workflow that actually happens: who does what, where data comes from, and which part jams up most often.

01 🔍

Audit the Workflow

  • Map how the work actually runs today
  • Find what's slow, duplicated, or scattered
  • Listen to the team that lives the workflow
  • Decide which part is most worth systemizing
Deliverables
Workflow map Bottleneck list Automation priorities
02 📐

Build the System

  • Turn the process into roles and access
  • Design the data structure and integrations
  • Automate only what's genuinely worth automating
  • Keep the rest easy for humans to control
Deliverables
Dashboard AI workflow Internal tool Admin panel
03 🚀

Make It Used, Not Just a Demo

Most systems fail not because the tech is bad, but because the team never adopts it. So I stay until it becomes part of the daily routine.

  • Ship dashboards, portals, and admin panels
  • Wire up AI assistants and automated reports
  • Train the team and document the system
  • Iterate based on what they actually use
Deliverables
Documentation Handover Team training Iteration
The End Result

What usually changes once the system is in place.

  • Admins no longer recap everything by hand each day
  • Owners see operational status straight from a dashboard
  • Customer follow-ups run automatically
  • Data stops scattering across many places
  • New hires onboard faster because the flow is documented
  • Reports generate automatically without waiting on one person

Tech I usually reach for: n8n, WhatsApp API, Vertex AI, PostgreSQL, Odoo, Python, SvelteKit, and more — chosen to fit the need, not the other way around.
